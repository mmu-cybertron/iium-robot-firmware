# Firmware Structure

This project keeps CubeMX-generated code separate from robot-owned firmware.

## Ownership

- `Core/` and `Drivers/`: generated by STM32CubeMX.
- `App/app_main.c` and `App/app_main.h`: robot application entrypoint.
- `App/Core/`: robot lifecycle, configuration, shared robot types.
- `App/BSP/`: direct STM32 HAL access and board mappings.
- `App/Drivers/`: hardware device drivers such as motors and sensors.
- `App/Services/`: interpreted robot signals such as edge detection and opponent tracking.
- `App/Control/`: motor commands and motion helpers.
- `App/Strategy/`: sumo behavior and state machine.
- `App/Utils/`: reusable helpers with no robot-specific policy.

## CubeMX Integration

Add this include to CubeMX `Core/Src/main.c`:

```c
#include "app_main.h"
```

Call `app_main()` after all `MX_*_Init()` functions:

```c
MX_GPIO_Init();
MX_I2C1_Init();
MX_TIM2_Init();
MX_TIM3_Init();

app_main();
```

Keep generated initialization in `Core/`. Put robot behavior in `App/`,
especially `App/Core`, `App/Services`, `App/Control`, and `App/Strategy`.

Add these include paths in STM32CubeIDE:

```text
App
App/Core/Inc
App/BSP/Inc
App/Drivers/Inc
App/Services/Inc
App/Control/Inc
App/Strategy/Inc
App/Utils/Inc
```
